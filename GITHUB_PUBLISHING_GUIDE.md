# Инструкция по публикации сайта на GitHub Pages

## Подготовка к публикации

1. Убедитесь, что у вас есть аккаунт на GitHub
2. Создайте новый репозиторий с именем в формате `ваше-имя-пользователя.github.io`
   - Например: `emily.github.io`
   - Замените `emily` на ваше имя пользователя на GitHub

## Публикация сайта

### Вариант 1: Через веб-интерфейс GitHub

1. Перейдите на сайт https://github.com и войдите в свой аккаунт
2. Нажмите кнопку "New repository" 
3. Введите имя репозитория в формате `ваше-имя-пользователя.github.io`
4. Не создавайте README, .gitignore или license - мы загрузим все файлы позже
5. Нажмите "Create repository"
6. На странице репозитория нажмите "Add file" → "Upload files"
7. Перетащите все файлы из папки `emily-website` в окно загрузки
8. После загрузки нажмите "Commit changes"
9. Перейдите в настройки репозитория (Settings)
10. Прокрутите вниз до раздела "Pages"
11. В секции "Source" выберите "Deploy from a branch"
12. Выберите ветку "main" и папку "/ (root)"
13. Нажмите "Save"
14. После нескольких минут ваш сайт будет доступен по адресу: `https://ваше-имя-пользователя.github.io`

### Вариант 2: Через Git команды

1. Установите Git на ваш компьютер
2. Откройте терминал или командную строку
3. Клонируйте репозиторий:
   ```
   git clone https://github.com/ваше-имя-пользователя/ваше-имя-пользователя.github.io.git
   ```
4. Скопируйте все файлы из папки `emily-website` в папку репозитория
5. Перейдите в папку репозитория:
   ```
   cd ваше-имя-пользователя.github.io
   ```
6. Добавьте файлы:
   ```
   git add .
   ```
7. Зафиксируйте изменения:
   ```
   git commit -m "Initial site deployment"
   ```
8. Загрузите на GitHub:
   ```
   git push origin main
   ```

## Настройки GitHub Pages

1. После загрузки файлов зайдите в настройки репозитория
2. Найдите раздел "Pages" 
3. Убедитесь, что выбрана ветка "main" и папка "/ (root)"
4. После сохранения настроек сайт станет доступен через несколько минут

## Проверка публикации

После успешной публикации ваш сайт будет доступен по адресу:
`https://ваше-имя-пользователя.github.io`

## Обновление сайта

Для обновления сайта просто измените файлы и закоммитьте изменения в ветку `main`.
GitHub Pages автоматически обновит сайт в течение нескольких минут.

## Примечания

- Файл `.nojekyll` в корне репозитория позволяет использовать папки с именами, начинающимися с подчеркивания
- Убедитесь, что все пути в HTML файлах корректны
- Для изображений и медиафайлов используйте относительные пути
